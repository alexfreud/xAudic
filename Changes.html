xAudic Change Log

0.21

  • now works on MacOS 10.1
  • supports filenames in other languages (Thanks to Shinichiro Shoji for the fixes)
  • adding a directory to the playlist really works now

0.20

  • New name: xAudic
  • drag/drop add directory works properly
  • added user interface plugin interface (see UICreation.html)
  • added some new user interfaces
  • added visualization plugin interface
  • added two new visualizations (ported from xmms)
  • now link all the built-in plugins directly instead of using bundles, faster load
  • better visualization performance
  • Equalizer can be now shaded
  • lots of fixes to moving, shading, and restoring the WinAmp UI windows
  • fixed problem with invalid MP3 files crashing the application

0.13

  • fixed sizing of file info box
  • file info can now show multiple values for one tag
  • fixed problem with wrong id3 tag being used in the playlist
  • http streaming working again
  • application version number now set
  • implemented playlist randomize, reverse and sort
  • made space key pause

0.12

  • EQ and Playlist windows will now stay hidden
  • windows don't pop to the front at odd times
  • playlist window now updates when a directory is added
  • fixed crash when non mp3's are added to the playlist
  • fixed time display
  • visualizations now synced with sound output
  • song position slider now works
  • ends of songs no longer get cut off
  • fixed volume slider

0.10

  • fixed problem with song titles not showing up correctly in the playlist.
  • updated the mp3 decoder, 160Kbps mp3's now work

0.8

  • works on MacOS X (MacOS X Server no longer supported)
  • new file icons from Donald Tripton
  • moving the application to another directory will not break the skin preference if the skin is in the app wrapper (from Todd Thomas rtthomas@mediaone.net).
  • bogus files will not show up in the skin browser (from Todd Thomas rtthomas@mediaone.net)
  • support for .wsz skin files
  • fixed minor skin issues

0.7

  • fixed bugs with some mp3's crashing or hanging the app
  • some ID3v2 tags would crash id3lib

0.6

  • fixed bug which caused some files either crash the app, or hang it up after playing
  • added ID3 v2.3 support
  • the visualization on the playlist window would not go away then the window was too small
  • added alert box when no input plugin can be found for a filen
  • removed mpg123 and mpeg3play input plugins
  • default filename extension preference now works
  • now displays bitrate for VBR files correctly
  • fixed another playlist save bug
  • ported new effects plugins from x11amp (Echo, Stereo, and SurroundEcho)

0.5
